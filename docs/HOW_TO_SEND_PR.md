# 🍔 QuickBite – How to Send a Pull Request (PR)

This guide is only for contributors who want to **fork the repository and submit a Pull Request (PR)**.

---

## 🚀 Step-by-Step: Send a PR

### 1. Fork the Repository

* Go to the QuickBite GitHub repo
* Click the **Fork** button (top right)
* This creates your own copy of the project

---

### 2. Clone Your Fork

```bash
git clone https://github.com/your-username/QuickBite.git
cd QuickBite
```

---

### 3. Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

✅ Use clear names like:

* `feature/add-cart`
* `fix/login-bug`
* `ui/improve-navbar`

---

### 4. Make Your Changes

* Add features / fix bugs / improve UI
* Keep changes small and focused

---

### 5. Commit Your Changes

```bash
git add .
git commit -m "Add: short clear description"
```

---

### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

---

### 7. Open a Pull Request

* Go to your fork on GitHub
* Click **Compare & pull request**
* Add:

  * A clear **title**
  * A short **description** of your changes
* Click **Create pull request**

---

## 💡 Quick Tips

* Do NOT push directly to `main`
* Always use a new branch
* Keep PRs small and clean
* Write clear commit messages
* Add screenshots for UI changes

---

## ❤️ Final Note

That’s it! Your Pull Request is now ready 🚀

Thank you for contributing to **QuickBite**!
